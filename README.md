# Geared AI 브라우저 자동화 도구 — 범용 사이트 크롤링 크롬 익스텐션

자연어로 "이 페이지에서 무엇을 추출/조작할지"를 적으면, 선택한 **AI 제공자(DeepSeek 또는 OpenAI)** 가
크롤링 **액션 스펙(JSON)** 을 생성하고, 익스텐션에 내장된 **고정 인터프리터**가 그 스펙을
실행해 데이터를 추출합니다. 결과는 **CSV**(+선택 PNG)로 저장됩니다.

> 설계: LLM이 날 JS를 실행하는 대신 **구조화된 액션 스펙**을 만들고, 검증된 인터프리터가
> 실행합니다. MV3 CSP에 걸리지 않고, 모든 사이트에서 동작하며, 임의 코드 실행 위험이 없습니다.

## 설치

1. `chrome://extensions` → **개발자 모드** ON
2. **압축해제된 확장 프로그램 로드** → `ai-crawler` 폴더 선택
3. (시크릿 모드 사용 시) 카드 **세부정보** → **시크릿 모드에서 허용** ON

## 프로젝트 관리 (사이트 + 프롬프트 + 액션 스펙)

이 익스텐션은 **프로젝트** 단위로 작업을 저장·재사용합니다. 한 프로젝트는
**시작 페이지(URL) · 사이트(도메인) · 프롬프트(자연어 작업) · 액션 스펙 · 옵션(스크린샷/최대 페이지)** 를 묶습니다.

- **목록 화면**: 저장된 프로젝트 카드 목록. **현재 탭의 사이트와 일치하는 프로젝트는 상단에 강조**됩니다.
  `+ 새 프로젝트`로 새로 만들고, 카드의 🗑로 삭제합니다. ⚙️ API 키 설정도 여기에 있습니다.
- **디테일 화면**: 프로젝트 이름 · **시작 페이지** · 프롬프트 · `①계획 생성` · 스펙 표시/수정 · `②실행` · **`프로젝트 저장`**.
- **시작 페이지**: 계획 생성·실행 시 **먼저 이 URL로 이동**한 뒤 동작합니다. 따라서 어떤 탭에서
  실행하든 프로젝트가 지정한 페이지에서 시작합니다(자기완결). `현재 탭` 버튼으로 지금 주소를 채울 수 있고,
  비워 두면 현재 열린 탭에서 그대로 동작합니다. 사이트(자동매칭 키)는 시작 페이지의 도메인으로 정해집니다.
- **자동 선택**: 아이콘을 누른 시점에 현재 사이트와 일치하는 프로젝트가 **정확히 1개**면
  그 프로젝트 디테일이 **바로 열립니다**(곧장 ②실행 가능). 0개거나 여러 개면 목록을 보여줍니다.
- **명시적 저장**: 프롬프트/스펙/옵션을 바꿔도 `프로젝트 저장`을 눌러야 보관됩니다(자동 저장 아님).
  저장하지 않은 변경이 있으면 `● 저장 안 됨`이 표시됩니다.

> 프로젝트는 `chrome.storage.local`에 저장됩니다(이 브라우저 한정). 저장 전(`+새 프로젝트` 직후)에는
> 목록에 보이지 않으며, 저장하지 않아도 ②실행은 동작합니다(다만 다음에 재사용 불가).

## 사용법

1. 크롤링할 **웹페이지를 직접 연다** (이 익스텐션은 *현재 활성 탭*을 대상으로 동작)
2. 익스텐션 아이콘 클릭 → **목록 화면**(또는 일치 프로젝트가 1개면 디테일로 자동 진입)
3. 처음 한 번 목록의 **⚙️ AI 제공자 · API 키 설정**에서 **DeepSeek/OpenAI 중 선택**하고
   해당 키 입력 → 저장 (제공자별로 이 브라우저에만 저장됨)
4. `+ 새 프로젝트` → **프로젝트 이름**, **시작 페이지**(기본값=현재 탭 주소, `현재 탭` 버튼/직접 입력),
   **할 작업**을 자연어로 입력. 예:
   - `상품 목록에서 상품명, 가격, 평점, 링크를 모두 추출해줘`
   - `검색창에 "노트북" 입력하고 검색한 뒤, 결과의 제목과 가격을 추출. 3페이지까지`
5. (선택) 화면 스크린샷 저장 체크, 최대 페이지 수 설정
6. **① 계획 생성** → DeepSeek가 **액션 스펙(JSON)** 을 만들어 팝업에 보여줍니다.
   스펙을 눈으로 검토하고, 셀렉터가 어긋났으면 **그 자리에서 직접 수정**할 수 있습니다.
7. **프로젝트 저장** → 이 사이트의 프롬프트·스펙이 저장되어 다음에 재사용됩니다.
8. **② 실행** → 검토(수정)한 스펙을 실행합니다. 다운로드 폴더에
   `crawl_<제목>_<시간>.csv` 생성 (결과 JSON 파일은 생성하지 않습니다)
9. 다음부터는 같은 사이트에서 아이콘만 누르면 프로젝트가 자동 선택되어 **바로 ②실행**할 수 있습니다.

> **계획과 실행이 분리되어 있습니다.** 작업 문구를 바꾸면 기존 스펙이 무효화되므로
> 다시 **①계획 생성**을 눌러야 합니다. LLM이 즉시 실행하지 않으므로, 잘못된 계획으로
> 헛도는 것을 막고 실행 전에 검증할 수 있습니다.

## 액션 스펙(DSL) — 인터프리터가 이해하는 동작

DeepSeek는 아래 액션들로 구성된 JSON을 생성합니다:

| action | 설명 |
|--------|------|
| `type` | `{selector, text}` 입력창에 텍스트 입력 |
| `click` | `{selector}` 요소 클릭 |
| `waitFor` | `{selector, timeoutMs}` 요소가 나타날 때까지 대기 |
| `wait` | `{ms}` 고정 시간 대기 |
| `scrollToBottom` | 끝까지 스크롤(lazy-load 유도) 후 맨 위 복귀 |
| `extract` | `{item, fields}` 각 항목(item)에서 fields 추출 |
| `paginate` | `{nextSelector, maxPages}` 다음 페이지로 넘기며 extract 반복 |

`fields` 값은 `item` 기준 상대 CSS 셀렉터이며, 속성 추출은 `a@href`, `img@src` 형식 사용.

> 📖 스펙 규격·문법·예제·디버깅은 **[ACTION_SPEC_GUIDE.md](ACTION_SPEC_GUIDE.md)** 참고.

```jsonc
{
  "steps": [
    { "action": "scrollToBottom" },
    { "action": "extract", "item": ".product",
      "fields": { "name": ".title", "price": ".price", "link": "a@href" } },
    { "action": "paginate", "nextSelector": "a.next", "maxPages": 3 }
  ]
}
```

## 동작 구조

| 파일 | 역할 |
|------|------|
| `popup.html/js` | **목록/디테일 2뷰**, 사이트 자동 선택, 프로젝트 저장, **①계획 생성 / 액션 스펙 표시·수정 / ②실행** |
| `store.js` | `chrome.storage.local` 기반 **프로젝트 CRUD**(load/upsert/delete/matchBySite) — popup.js보다 먼저 로드 |
| `background.js` | `AI_PLAN`(DOM 요약 → DeepSeek로 스펙 생성·반환) / `AI_EXECUTE`(받은 스펙을 **단계별 executeScript 주입 실행** → CSV 다운로드). 진행 상황은 `PROGRESS` 메시지로 팝업 모니터에 전달 |

> **메시지 2종으로 분리**: `AI_PLAN`은 스펙만 만들어 팝업에 돌려주고, `AI_EXECUTE`는
> 팝업이 (사용자가 검토·수정한) 스펙을 보내면 그대로 실행합니다.
>
> 액션 스펙은 background가 단계마다 `chrome.scripting.executeScript`로 페이지에 직접 주입해
> 실행합니다. content script에 통째로 위임하지 않으므로, 클릭·검색·페이지 이동으로 페이지가
> 새로 로드돼도 끊기지 않습니다("message channel closed" 에러 방지).

## 알아둘 점 / 한계

- **API 키 보관**: `chrome.storage.local`(이 브라우저)에 평문 저장됩니다. 개인용에 적합하며,
  공용 PC에서는 사용 후 키를 지우세요.
- **DOM 요약 한계**: 페이지 HTML을 약 16,000자로 잘라 LLM에 전달합니다. 매우 크거나
  복잡한 페이지는 셀렉터 정확도가 떨어질 수 있습니다.
- **셀렉터 빗나감**: 결과가 비면 보통 LLM이 만든 셀렉터가 안 맞은 것입니다. 작업 문구를 더
  구체적으로(예: "상단 검색결과 카드") 쓰거나, 다운로드된 `.json`의 `spec`을 보고 작업을 조정하세요.
- **SPA/지연 로딩**: 클릭 후 콘텐츠가 늦게 뜨면 작업 문구에 "로딩될 때까지 기다렸다가"를 명시하면
  `waitFor` 단계가 들어갑니다.
- **로봇 차단/약관**: 사이트의 robots.txt·이용약관을 존중하세요. 과도한 자동화는 차단될 수 있습니다.
- **비용**: 실행마다 DeepSeek API를 1회 호출합니다(토큰 비용 발생).
- **스크린샷**: 전체 페이지를 저장합니다. 기본은 CDP로 **한 번에 단일 이미지** 캡처이며, 이때
  상단에 "…디버깅을 시작했습니다" 배너가 잠깐 뜹니다(정상). 디버거 불가 시 자동으로 뷰포트
  여러 장 이어붙이기로 폴백합니다.
